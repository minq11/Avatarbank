## Buyer & Influencer 화면 설계

> 이 문서는 `Avatarbank`의 **실제 사용자(Buyer / Influencer)**가 사용하는 프론트엔드 화면을 정의한다.  
> 컴포넌트 구조, UX 흐름, 주요 상태값을 이해하기 위한 수준의 설계 문서이다.

---

## 1. 공통 영역

### 1.1 공통 레이아웃

- **헤더**
  - 좌측: 서비스 로고 / 이름 (`Avatarbank`)
  - 중앙: 메인 내비게이션 + 언어 콤보박스
    - 내비게이션:
      - `마켓` (아바타 목록)
      - `갤러리` (Public Gallery, 선택 사항)
    - 언어 선택 콤보박스:
      - 기본 값: `EN`
      - 옵션: `EN`, `KO`, `JA`
  - 우측:
    - 비로그인: `로그인`, `회원가입` 버튼
    - 로그인:
      - 크레딧 잔액 표시 (`100 C` 등)
      - 프로필 드롭다운
        - `내 아바타` (Influencer 전용)
        - `내 생성물`
        - `크레딧 충전`
        - `설정`
        - `로그아웃`

- **푸터**
  - 이용약관, 개인정보처리방침, 인플루언서 안내, 문의/지원 링크

---

## 2. 랜딩 / 온보딩

### 2.1 랜딩 페이지

- **목표**
  - 서비스 컨셉 소개
  - Buyer / Influencer 각각의 가치 제안 전달

- **구성**
  - 히어로 섹션
    - 메인 카피: “인플루언서 아바타로 나만의 이미지를 생성하세요”
    - 서브 카피: “이미지 1장마다 인플루언서와 수익을 나누는 AI 아바타 마켓플레이스”
    - CTA 버튼:
      - `지금 생성해보기` (마켓으로 이동)
      - `인플루언서로 참여하기` (인플루언서 온보딩 설명 페이지)
  - 샘플 갤러리 썸네일 (주요 아바타/결과 이미지 일부 노출)
  - 인플루언서 섹션
    - “사진만 올리면, AI 아바타가 수익을 만듭니다”
    - `수익 구조 보기` 링크 (크레딧/환전 설명 모달 또는 페이지)
  - 어떻게 동작하나요? (3단계 카드)
    - Buyer: `아바타 선택 → 프롬프트 입력 → 이미지 다운로드`
    - Influencer: `인증 → 사진 업로드 → 자동 수익`

---

## 3. 인증 / 계정

### 3.1 로그인 / 회원가입

- **로그인 모달/페이지**
  - 이메일 + 비밀번호 로그인
  - 소셜 로그인 (선택: `Google`, `Apple`, `GitHub` 등)
  - 비밀번호 찾기 링크

- **회원가입 페이지**
  - 기본 정보: 이메일, 비밀번호, 닉네임
  - 역할 선택:
    - `Buyer로 시작` (기본값)
    - `Influencer로 참여` (선택 시 이후 온보딩에서 인스타 인증 안내)

### 3.2 계정 설정 페이지

- 탭 구성:
  - `프로필`
  - `보안`
  - `알림`

- **프로필 탭**
  - 닉네임, 프로필 이미지, 자기소개

- **보안 탭**
  - 비밀번호 변경
  - 2FA (향후)

---

## 4. 마켓 (아바타 목록)

### 4.1 아바타 리스트 페이지

- **목표**
  - 다양한 인플루언서 아바타를 탐색하고, 하나를 선택해 생성 flow로 진입.

- **구성**
  - 상단 필터/검색 바
    - 검색: 인플루언서 이름, 태그, 설명 키워드
    - 필터:
      - NSFW 허용 여부 토글 (기본 OFF)
      - 정렬: 인기순, 최신순, 수익순
  - 아바타 카드 그리드
    - 프리뷰 이미지 썸네일
    - 아바타 이름 / 인플루언서 핸들
    - 간단 설명 (1~2줄)
    - 기본 생성 비용 정보 (1 Credit + 옵션)
    - 배지:
      - `NSFW 가능` (옵션)
      - `신규`, `인기`

- **상호작용**
  - 카드 클릭 → 아바타 상세 페이지로 이동

---

## 5. 아바타 상세 / 이미지 생성

### 5.1 아바타 상세 페이지

- **좌측 영역**
  - 큰 프리뷰 이미지
  - 샘플 이미지 썸네일 리스트 (스크롤)

- **우측 영역**
  - 아바타 이름, 인플루언서 정보
  - 수익 구조 요약:
    - “기본 1C는 플랫폼, 옵션 크레딧은 인플루언서에게 지급됩니다.”
  - NSFW 정책 안내 (허용/비허용, 주의 문구)

### 5.2 이미지 생성 패널

- **입력 요소**
  - 텍스트 에어리어: `프롬프트 입력` (placeholders로 예시 프롬프트 몇 개 제공)
  - 옵션:
    - 이미지 스타일 선택 (카드/셀렉트)
    - 해상도 프리셋 (기본값 512x768 등)
  - 옵션 크레딧 슬라이더
    - 최소 0 ~ 최대 10 (예시)
    - 실시간으로 “총 비용: X Credit” 계산 표시

- **요약 & CTA**
  - “현재 보유 크레딧: 120 C”
  - “이번 생성에 사용: 1C + 옵션 4C = 5C”
  - `이미지 생성` 버튼 (비활성 조건: 프롬프트 비어있음, 크레딧 부족)

### 5.3 생성 진행 / 결과

- **생성 중 상태**
  - 로딩 인디케이터 + 예상 소요 시간 표시
  - “생성 대기열에 등록되었습니다. 완료 시 알림으로 알려드립니다.”

- **성공 시**
  - 생성된 이미지 크게 표시
  - 하단 액션:
    - `다운로드`
    - `다시 생성` (동일 프롬프트/옵션으로 재시도)
    - `프롬프트 수정 후 재생성`
    - `공유하기` (shared_generations 연동)

- **실패 시**
  - 에러 메시지 (친절한 문구)
  - “크레딧이 차감되지 않았습니다” 또는 정책에 따른 안내
  - `다시 시도` 버튼

---

## 6. 내 생성물 / 공유

### 6.1 내 생성물 리스트

- **목표**
  - Buyer가 자신이 생성한 이미지를 한 곳에서 관리.

- **구성**
  - 필터:
    - 아바타별 필터
    - 생성 상태(성공/실패)
  - 카드:
    - 이미지 썸네일
    - 생성일시
    - 사용 크레딧
    - 아바타 이름
  - 카드 hover/클릭 시 액션:
    - 상세 보기
    - 다운로드
    - 공유 링크 생성/관리 (shared_generations)

### 6.2 공유 링크 관리 모달

- 공유 상태:
  - 공개 여부 토글 (`Public Gallery에 노출`)
  - 공유 URL 표시 + 복사 버튼
  - (선택) 만료 기간 설정

---

## 7. 크레딧 / 결제

### 7.1 크레딧 충전 페이지

- **상단 요약**
  - 현재 보유 크레딧
  - “5 Credit = 1 USD (구매 기준)” 안내 문구

- **충전 옵션 카드**
  - 예시:
    - 50 C / 10 USD
    - 120 C / 24 USD
    - 300 C / 60 USD
  - 각 카드에 `선택` 버튼

- **결제 플로우**
  - 선택 후 `PayPal Checkout` 모달/리다이렉트
  - 결제 완료 후 성공/실패 상태 표시
  - `최근 결제 내역` 섹션 (transactions 연동)

---

## 8. Influencer 전용 화면

### 8.1 인플루언서 대시보드

- **상단 요약 카드**
  - 누적 수익 (크레딧/USD)
  - 이번 달 수익
  - 총 생성 횟수

- **중앙**
  - 아바타별 수익/생성 횟수 차트
  - 최근 생성 프롬프트/이미지 일부 리스트 (정책 허용 범위 내)

### 8.2 아바타 관리 페이지

- 탭:
  - `내 아바타`
  - `새 아바타 등록`

- **내 아바타**
  - 아바타 카드 리스트:
    - 상태 배지 (승인 대기, 활성, 거절, 숨김)
    - NSFW 허용 여부
    - 프리뷰 이미지
    - `상세 설정` 버튼

- **새 아바타 등록 플로우**
  1. 인스타그램 계정 인증 상태 확인 (미완료 시 인증 안내)
  2. 필수 사진 업로드 (정면, 좌/우, 전신)
  3. 아바타 이름/설명 입력
  4. NSFW 허용 여부 설정
  5. 제출 → “Admin 승인 대기” 상태 표시

### 8.3 수익 / 환전 페이지

- **상단**
  - 현재 환전 가능 크레딧
  - 환전 비율 안내: `10 Credit = 1 USD`

- **환전 요청 폼**
  - 환전 신청 크레딧 입력 (최소 기준 안내)
  - 예상 USD 금액 자동 계산
  - PayPal 계정 정보 확인/수정
  - `환전 요청` 버튼

- **환전 내역 리스트** (payout_requests 연동)
  - 요청일, 크레딧, USD, 상태(pending/paid/failed 등), 비고

## 9. 국제화 / 다국어(i18n) 관련 UI

- **기본 언어 및 타깃**
  - 기본 타깃 사용자는 **영어권**으로 가정한다.
  - 프론트엔드의 **기본 표시 언어(Default Locale)는 영어(EN)**로 한다.

- **언어 토글**
  - 헤더 우측 또는 설정 메뉴에 **언어 선택 토글(EN / KO / JA)** 제공.
  - 초기 진입 시:
    - 로그인 사용자는 **계정에 저장된 선호 언어(locale)**를 우선 사용.
    - 비로그인 사용자는 **브라우저 언어 감지 → 지원 언어(EN/KO/JA) 중 매핑 → 기본은 EN**.

- **텍스트 리소스 관리**
  - 모든 UI 텍스트는 **키 기반 리소스 파일**로 분리 (예: `en.json`, `ko.json`, `ja.json`).
  - 주요 시스템 메시지(에러, 토스트, 버튼 라벨 등)는 **3개 언어 모두 제공**을 목표로 한다.

- **프롬프트 및 콘텐츠**
  - 프롬프트 입력 영역은 **언어 제한 없이 자유 입력**을 허용한다.
  - 다만, 안전/정책 필터링은 Backend의 정책에 따라 **주로 영어 기준(또는 다국어 지원 필터)**를 사용한다.
  - 마켓/갤러리에서 노출되는 기본 설명 문구는 **영어를 기본으로 작성**하고, 필요 시 한/일 번역본을 추가하는 방향으로 UI 구성한다.

> 이 문서는 **화면 목록과 주요 요소/플로우 수준의 설계**이며,  
> 실제 컴포넌트 구조/라우팅 설계는 이후 `프론트엔드 아키텍처 설계 문서`에서 다룬다.
